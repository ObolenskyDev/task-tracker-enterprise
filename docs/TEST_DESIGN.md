# Тест-план: Функционал создания задач (SCRUM-2)

## 1. Тест-кейсы (Test Cases)

| ID | Заголовок | Шаги | Ожидаемый результат |
|--- |--- |--- |--- |
| **TC-01** | Успешное создание задачи (Позитивный) | 1. Отправить POST /api/v1/tasks<br>2. Body: `{"title": "Test Task"}` | Код 201 Created. Задача появилась в БД. |
| **TC-02** | Валидация пустого заголовка (Негативный) | 1. Отправить POST /api/v1/tasks<br>2. Body: `{"title": ""}` | Код 400 Bad Request. Ошибка валидации. |

## 2. Граничные значения (Boundary Values)
Согласно схеме БД, поле `title` имеет лимит 200 символов.

* **Значение 1:** 1 символ (Минимум) -> Успех (201)
* **Значение 2:** 200 символов (Максимум) -> Успех (201)
* **Значение 3:** 201 символ (Выход за границу) -> Ошибка (400 или 500)
